# 프로젝트 기여 가이드라인 V1.0

## *필수* :  Asset 및 Prefab 관리 규칙

원활한 협업과 데이터 손실 방지를 위해 다음 규칙을 따릅니다.

### 1. 원본 불변의 법칙 (Immutable Originals)
* **규칙:** 타인이 작성하거나 구매/다운로드한 원본 에셋(스크립트, 프리팹, 머티리얼 등)은 직접 수정하지 않습니다.
* **사유:** 원본이 수정되면 해당 원본을 참조(Reference)하는 다른 팀원의 작업물에서 의도치 않은 변경이나 오류가 발생합니다.

### 2. 수정 절차 (Workflow)
수정이 필요한 경우 다음 절차를 따릅니다:
1.  Project 창에서 수정하려는 원본 파일을 선택합니다.
2.  `Ctrl + D`를 눌러 파일을 복제합니다.
3.  복제된 파일의 이름을 변경합니다 (예: `OriginalName_Fixed_YourName`).
4.  복제된 파일을 수정하여 사용합니다.


## *권장* : 공용 파일 및 각 기여자 파일 생성

다음은 위와같은 충돌 및 치명적 오류를 방지하기 위해 권고하는 사항입니다.

### 1. Main과 Dev_(기여자 이름)으로 물리적 파일 분리
* **방식:** 공용으로 사용되는 에셋과 프리펩은 Main에 배정합니다. 그리고 각 기여자는 Dev_()파일을 만들어 자유롭게 테스트 및 수정을 하되, 다른 사람의 Dev 폴더나 Main 폴더에 있는 것은 건드리지 말고, 필요하면 내 폴더로 복사해 와서 쓰라고 공지합니다.
